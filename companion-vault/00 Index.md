# Boss Cat / Companion Vault

This is an Obsidian vault (just open this folder in Obsidian). It documents the
**custom additions** built on top of the upstream [clawd-on-desk](https://github.com/rullerzhou-afk/clawd-on-desk)
project for Ilia -- not the upstream app itself. Upstream architecture already
has its own docs in `docs/project/` (see `AGENTS.md` at the repo root); this
vault only covers what's new.

Read **this note first**, then only open the specific note you need. That's
the point of the vault: a future Claude Code session (or you) should be able
to read one short note instead of re-reading the source files to remember how
something works.

## What was added, in one paragraph

A custom theme called **Boss Cat** (cream cat in a business suit with a
briefcase) replaces the default crab. A companion script,
**companion-watcher.js**, runs alongside Clawd and watches for real games,
music, and video playing on the PC (not just coding-agent activity), and
drives Boss Cat's animations live via Clawd's existing local hook API -- no
core app logic needed to change for that part. Two small additive patches to
core files make Discord Rich Presence show the right thing during those
activities too.

## Notes in this vault

- [[Local Hook API Cheatsheet]] -- the `POST /state` contract: what fields
  exist, which values are valid, how sessions live/die. This is the thing
  that took the most exploration to reverse-engineer from source; read this
  before touching anything that talks to Clawd over HTTP.
- [[Boss Cat Theme]] -- theme.json structure, the 11 SVG states, and how the
  `displayHintMap` mechanism lets one logical state ("juggling") show three
  different pieces of art depending on what's happening.
- [[Companion Watcher]] -- how companion-watcher.js decides what you're
  doing (games via process list, music/video via Windows SMTC), the
  start/switch/stop session lifecycle, and how to extend it.
- [[Discord Presence Patch]] -- the two small diffs to core files
  (`state-session-snapshot.js`, `discord-presence-rpc.js`) that make Discord
  show "Playing a game" / "Listening to music" / "Watching a video" instead
  of the generic "Working" bucket.
- [[Changelog]] -- dated record of what changed and why (round 1).
- [[Feature Pass 2]] -- meetings, streaks, break reminders, daily/weekly
  stats, and sound stings (round 2).
- [[Auto-Start]] -- how Clawd and companion-watcher.js both launch on system
  startup without manual commands: reused the existing "Open at
  login" toggle for Clawd itself, and a new `companion-watcher-sidecar.js`
  that spawns the watcher alongside Clawd's own lifecycle.
- [[Feature Pass 3]] -- Discord presence redesign (title=activity,
  hint=auto-tracked), cross-platform hardening (better mac/Linux media
  detection), AFK detection, combined-activity stats, lifetime achievements,
  configurable break threshold, Telegram phone alerts, the new Settings >
  Companion tab, and a packaged-build fix.
- [[Feature Pass 4]] -- rotating flavor quips, a rare idle easter egg, a
  full level/XP progression system with playful titles, and level-gated
  cosmetic accessories (party hat, sunglasses) equippable from Settings.
- [[Companion Stats]] -- auto-generated by `npm run companion-report`; your
  actual streak/stats data, not documentation.

## File map

```
clawd-on-desk/
  scripts/
    companion-watcher.js     <- the watcher (games + media), auto-started by Clawd (see [[Auto-Start]])
    media-status.ps1         <- Windows SMTC query, called by companion-watcher.js
    games.config.json        <- editable list of game process names to react to
  src/
    state-session-snapshot.js  <- PATCHED: added `displayHint` field to session snapshot entries
    discord-presence-rpc.js    <- PATCHED: companion-session special case in buildPresencePayload()
    companion-watcher-sidecar.js  <- NEW: spawns/restarts/stops companion-watcher.js as a child process
    main.js                    <- PATCHED: starts/stops the sidecar with Clawd's own lifecycle
  companion-vault/            <- this Obsidian vault
themes/  (or %APPDATA%/clawd-on-desk/themes/ once installed)
  boss-cat/
    theme.json
    assets/*.svg               <- 11 hand-authored SVGs with CSS animations
```

Both source patches are additive (new field, new early-return branch) -- they
don't change behavior for any existing coding-agent session, so `git pull`
from upstream stays low-risk to re-apply if it ever conflicts.
